## Problem

A real estate business needed a professional web presence with manageable property listings, education content, and lead capture — without developer involvement for every listing update.

## Goal

Build a production-grade site where non-technical staff edit content in Sanity, and the Next.js frontend handles SEO, i18n, and enquiry flows.

## Shipped vs remaining

**Shipped today:**
- Property schema and Sanity Studio
- Homepage, listings, property detail routes
- Contact API + Resend integration
- Newsletter API endpoint
- i18n routing structure

**Remaining:**
- Content population and launch polish
- SEO metadata refinement per property
- Performance audit and image optimization pass

## Why Next.js (not Astro)

This is a **client production site** — it needs Sanity preview, dynamic listings, and ongoing content edits. The portfolio site (Astro) is a separate concern: static, fast, MDX-driven. Using the right tool per job matters.
