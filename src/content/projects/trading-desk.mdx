## Problem

Recurring execution patterns (scratch cycles, bias lock, early breakeven stops) were identified in reviews but not **interrupted in session**. Notes alone don't change behaviour — you need an operating system at the point of decision.

## Goal

Build a process framework — not a signal service — that helps detect schema activation, run state checks, and apply documented interventions before damage compounds.
